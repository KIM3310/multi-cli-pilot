# Monetization Playbook - multi-cli-pilot

This file is a commercial planning artifact, not a guarantee of revenue. Use it to make the repository legible as a sellable product, pilot, workshop, license, or proof-of-value package.

## Commercial Lane

- **Lane:** B2B developer productivity
- **Primary buyer:** Engineering teams and platform groups standardizing multiple coding-agent CLIs.
- **Revenue wedge:** One orchestration harness for Gemini, Qwen, prompts, workflows, hooks, MCP, and team primitives.

## Offer Ladder

1. **Internal workflow setup** - pricing hypothesis: $2k-$8k setup
2. **Team-agent playbook** - pricing hypothesis: $10k-$35k team rollout
3. **MCP orchestration starter** - pricing hypothesis: $1k-$6k/month workflow maintenance

## Acquisition Channels

- Developer productivity demos
- CLI workflow templates
- Engineering manager outreach

## Proof Gates Before Selling

- npm run verify passes
- Installers work
- Workflow examples are inspectable

## Risk Boundaries

- Human review remains required
- Secrets stay local
- Generated code needs CI gates

## 30/60/90 Day GTM

- **30 days:** Publish 5 workflow recipes
- **60 days:** Create team rollout guide
- **90 days:** Offer setup sprint

## Metrics That Matter

- Workflow completion
- Review cycle time
- Failed-agent recovery rate

## Positioning Note

Lead with the specific operational pain, then show the runnable proof, then offer a narrow pilot. Do not pitch the repository as a vague app; pitch it as a concrete workflow with evidence, risk boundaries, and a next buying step.
